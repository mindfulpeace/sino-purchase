## P0（安全 + 正确性，已止血）

### P0-2 凭据移出源码 → 环境变量
- `apps/sino-purchase-v2/src/config/sheets.ts`：改读 `import.meta.env.VITE_CLIENT_ID` / `VITE_SPREADSHEET_ID`，缺失时降级演示模式并告警。
- 新增 `apps/sino-purchase-v2/.env`（gitignored，已填入真实值，本地开发不受影响）与 `.env.example`（占位模板）。
- `vite-env.d.ts` 增加 `ImportMetaEnv` 类型声明。
- `SheetsEditor.tsx` / `SheetsDataTab.tsx`：ID 为空时不再渲染坏链/坏 iframe，改为友好提示。

### P0-1 OAuth 令牌不落盘 + 加 CSP
- `packages/sheets-core/src/auth.ts`：移除 `localStorage` 明文令牌，令牌仅存内存（`accessToken`/`expiresAt` 模块变量），登录态在刷新后由 GSI 静默重认证恢复。userinfo 仍缓存（非敏感）。
- `apps/sino-purchase-v2/index.html`：新增严格 `Content-Security-Policy`（限制 `script-src` 仅 `self` + `accounts.google.com`，`connect-src` 仅 Google API 域，`frame-src` 仅 Google Sheets/账户），从源头消除 XSS 窃取令牌的向量。
- 备注：scope 收窄（限制到单表）需后端代理持有令牌，留作后续演进。

### P0-3 修复云端数据漂移（架构正确性 bug）
- `packages/sheets-react/src/useSheetData.ts`：缓存 `loadTable` 返回的 `rowMap`（id→行号）；`update`/`remove` 直接用行号写，**不再每次 `findRow` 全列扫描**。
- `remove` 失败路径现会把 `rowIndex` 一并入队（原代码漏传 rowIndex → 永久静默丢失），`processQueue` 据此能真正在云端删除。

### P0-4 同步队列按错误类型分流 + 超时
- `packages/sheets-core/src/db.ts`：新增 `SheetsApiError` 与 `isRetryableStatus()`。网络中断/超时、429、5xx 可重试；**401（无 refresh 只能重登）与 4xx 业务错误判为永久失败并移出队列**，停止无限重排到 1000 条上限。
- `fetchWithAuth` 增加 `AbortController` 15s 超时。
- `sync-queue.ts` 新增 `SYNC_ERROR_EVENT`，永久失败时广播；`useSync` 暴露 `failedCount` 供 UI 提示。

## P1（质量 + 性能，提升健壮性）

### P1-5 移除 `// @ts-nocheck`
- `CashGrid.tsx`、`MaterialInfo.tsx` 移除 `// @ts-nocheck`，类型检查全部通过（两者导出 API 与 store 签名均匹配）。
